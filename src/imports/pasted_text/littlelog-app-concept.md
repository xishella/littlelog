Design a warm, friendly, human-centered mobile app MVP for a baby caregiver handoff app called LittleLog.

LittleLog helps caregivers quickly log a baby’s day, including sleep, bottles, nursing, diapers, solids, mood, notes, and photos. At the end of a caregiver session, the app uses AI to turn the day’s logs into a clear, parent-friendly handoff summary. The product should feel soft, caring, modern, and helpful, not clinical or overly “babyish.”

Brand direction:
- Keep the existing LittleLog color palette from the brand board.
- Use a warm cream/off-white background as the main app base.
- Use the deeper brand color for the opening screen and key hero moments.
- Use soft pastel card colors throughout, inspired by baby care, mood tracking, and human-centered wellness apps.
- Visual style should feel soft, warm, rounded, calm, and intentional.
- Avoid sterile medical UI, harsh blues, sharp corners, or overly corporate layouts.
- Use subtle abstract vector shapes, pebble-like forms, tiny leaf/baby/nature-inspired details, and gentle illustrative accents.
- The app should feel like a cozy daily handoff, not a hospital tracker.

Typography:
- Use Nunito for body text and general UI text.
- Use TT Hoves for headlines, subheadings, navigation labels, and card titles.
- Use The Seasons sparingly for brand moments, expressive headers, and warm title moments only.
- Serif typography should be used lightly to add warmth, not everywhere.

Design inspiration:
- From the mood tracker example: use a rich colored hero area with soft abstract vector shapes and friendly greeting copy like “Good evening” or “How was Zayd’s day?”
- From the soft parenting app examples: use rounded cards, pastel blocks, friendly illustrations, and cozy mobile dashboard layouts.
- From the progress dashboard example: include a soft bubble-style completion visualization that represents how ready the handoff is.
- From the baby tracker example: include simple activity tracking, but make it feel more polished, warm, and less generic.

Create 6 iPhone mobile screens:

1. Welcome / opening screen
- Deep brand color background with subtle abstract shapes.
- Greeting copy: “Good evening, Maya”
- Main headline: “How was Zayd’s day?”
- Supporting copy: “A simple handoff for the little moments, big updates, and everything in between.”
- Primary CTA: “Start today’s log”
- Use The Seasons sparingly for the warmest brand/title moment.

2. Today dashboard
- Warm cream background.
- Header with baby name, date, and caregiver profile/avatar.
- Top card: “Today with Zayd”
- Soft stat cards for Sleep, Bottles, Diapers, Solids, Mood, and Notes.
- Recent activity timeline with logged items.
- A persistent bottom navigation with Home, Log, Summary, Profile.
- Use rounded pastel cards and soft shadows.

3. Quick log screen
- Header: “What would you like to log?”
- Large rounded cards/buttons for Sleep, Bottle, Nursing, Diaper, Solid food, Medicine, Mood, Note, and Photo.
- Each card should have a simple friendly icon or small illustration.
- Make the layout fast and caregiver-friendly.
- Include a floating “+” action or soft primary CTA.

4. Log detail screen
- Create one detailed example for “Log a bottle.”
- Fields: Amount, Time, Type, Notes.
- Type options: Breast milk, Formula, Mixed.
- Use large touch-friendly controls and soft form cards.
- Primary CTA: “Save log”
- Keep the form simple and calming.

5. End session / handoff readiness screen
- Header: “Ready to wrap up?”
- Supporting copy: “LittleLog can turn today’s logs into a simple handoff summary for Zayd’s parent.”
- Include a soft bubble-style completion visualization inspired by the reference image.
- Show “Handoff ready: 89%”
- Include a checklist of logged categories: Sleep, Feeding, Diapers, Mood, Notes, Important updates.
- Primary CTA: “Create handoff summary”
- Secondary CTA: “Add one more note”

6. AI handoff summary screen
- Header: “Today’s handoff for Zayd”
- Supporting copy: “Generated from today’s logs. Review before sharing.”
- Large editable summary card with warm AI-generated copy.
- Sections: Quick summary, Sleep, Feeding, Diapers, Mood, Important notes.
- Include buttons: “Edit summary” and “Share with parent.”
- Make the AI feel subtle, helpful, and human, not futuristic or robotic.

UX tone:
- Copy should sound warm, natural, reassuring, and clear.
- Avoid clinical language.
- Avoid overly cutesy baby talk.
- Use phrases like:
  “Little moments logged.”
  “Ready when you are.”
  “Here’s how the day went.”
  “Anything parent should know?”
  “Create handoff summary.”

UI requirements:
- iPhone app design.
- Rounded cards, generous spacing, soft shadows.
- Warm cream base background.
- Pastel cards for activity categories.
- Deep brand color for high-emphasis areas.
- Soft iconography.
- Friendly, readable hierarchy.
- Clean enough to eventually build in React/Next.js.
- Keep the MVP focused and not overloaded.

Deliver the screens as a cohesive mobile app concept with consistent spacing, typography, color usage, navigation, card styles, and component patterns.
